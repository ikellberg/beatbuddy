# Plans

Здесь хранятся планы для каждой User Story.

## Структура

Каждый план создаётся перед началом работы над story и содержит:
- Декомпозицию на vertical slices
- Технические решения
- Критические файлы для модификации
- План верификации

## Процесс

1. Перед началом story запускается **Plan Mode**
2. План сохраняется в `docs/plans/US-XXX-название.md`
3. После утверждения плана начинается реализация
4. План используется как reference во время разработки

## Naming Convention

Формат: `US-XXX-короткое-название.md`

Примеры:
- `US-001-setup-metronome.md`
- `US-002-metronome-engine.md`
- `US-003-keyboard-sensor.md`
