## Код-ревью: US-017 — Счётчик streak успешных попаданий

### Вердикт: Требует доработки

### Невыполненные Acceptance Criteria

- **AC: "Streak корректно сбрасывается при старте новой сессии"** — реализовано частично. В `src/web/js/app.js` сброс вызван только после завершения countdown, поэтому при повторном старте во время 3-2-1 на экране может кратко показываться streak из прошлой сессии.

### Замечания к коду

1. **[БЛОКЕР]** **`src/web/js/app.js`, строка 371**: сброс streak происходит слишком поздно — после countdown.
   - Сейчас: `updateStreakDisplay(0)` вызывается после `await countdown(3)`, при этом `sessionScreen` уже показан на строках 353-355.
   - Нужно: вызывать `updateStreakDisplay(0)` сразу после показа Session Screen и до `await countdown(3)`.
   - Причина: нарушается AC про корректный сброс streak на старте новой сессии; пользователь видит устаревшее значение из предыдущей попытки.

2. **[ЗАМЕЧАНИЕ]** **`src/web/index.html`, строка 51**: в Session UI добавлен только текущий streak, но нет отдельного отображения best streak в той же сессии.
   - Сейчас: есть `#streak-display` (текущая серия), а `bestStreak` выводится только на Stats Screen (`#stats-best-streak`, строка 93).
   - Нужно: либо добавить best streak в Session Screen, либо явно зафиксировать в стори/technicalNotes, что best streak показывается только в итоговой статистике.
   - Причина: в `technicalNotes` US-017 указано "Добавить Session UI-элемент для combo и best combo"; текущая реализация этому не полностью соответствует.
